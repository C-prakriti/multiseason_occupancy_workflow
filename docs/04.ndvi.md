# Preparing NDVI covariate for Multiseason Occupancy analysis

## Objectives

To obtain NDVI data for three years 2017, 2018 and 2021 in order to
analyze the prey index and use as a site-level covariate in occupancy
analysis

## Data Source

NDVI data were obtained from the MODIS vegetation Indices product
available in Google Earth Engine. Dataset: - MODIS/061/MOD13Q - Product
name: MOD13Q1 Version 6.1 - Provider: NASA LP DAAC - Spatial resolution:
250 m - Temporal resolution: 16-day composite - Temporal coverage: 2017,
2018, 2021 - Source platform: Google Earth Engine - Images with high
cloud cover were filtered out prior to NDVI calculation. Median NDVI
composites were generated for each study year.

## NDVI Formula

NDVI is calculated as: NDVI = (NIR - red)/(NIR+Red) Here, NIR = Near
infrared band Red = red reflectance band

## Workflow Overview

-   Access MODIS MOD13Q1 NDVI image collection in Google Earth Engine
-   Filter imagery by study area
-   Generate annual NDVI composites
-   Scale MODIS NDVI values
-   Import sampling sites FeatureCollection
-   Extract mean NDVI values for each site usign reduceRegions()
-   Export site-level NDVI tables as csv files
-   Use extracted values as occupancy covariates in R

## Detailed Workflow Steps

### Step 1: Access MODIS NDVI Dataset in GEE

The MODIS MOD13Q1 Version 6.1 image collection ('MODIS/061/MOD13Q1') was
accessed through Google Earth Engine. The MOD13Q1 product provides
precomputed vegetation indices, including NDVI, at 250 m spatial
resolution with a 16-day temporal composite.

`var ndviCollection = ee.ImageCollection("MODIS/061/MOD13Q1")`

### Step 2: Filter images by study area and time period

The image collection was filtered to the study area boundary and the
study period corresponding to the camera trapping survey years. Only
images within the selected temporal range were retained for analysis.

### Step 3: Select NDVI bands

The NDVI band was selected from the MOD13Q1 image collection for
subsequent processing and making composites. The MOD13Q1 product
contains NDVI values scaled by a factor of 0.0001.

`var ndvi_2017 = ndviCollection .filterBounds(Grid) .filterDate('2016-11-01', '2017-02-28') .select('NDVI'); print(ndvi_2017);`

### Step 4: Generate NDVI composite raster

Mean NDVI composites were generated from filtered image collection to
represent average vegetation productivity during the study period.

`var ndvi_mean_2017 = ndvi_2017.mean();`

### Step 5: Scaling NDVI values

NDVI values from the MOD13Q1 product were scaled using the dataset scale
factor(0.0001) to convert stored integer values into standard NDVI
values ranging from -1 to +1.

`var ndvi_scaled_2017 = ndvi_mean_2017.multiply(0.0001);`

### Step 6: Import sampling locations

Sampling site locations were imported into Google Earth Engine as a
FeatureCollection for site-level NDVI extraction

`var points = ee.FeatureCollection("projects/inspired-bus-428412-c0/assets/GRID");`

### Step 7: Extract NDVI values to Sites

Mean NDVI values were extracted for all sampling sites using the
'reduceRegions()' function in Google Earth Engine. The extraction
used: - sampling site FeatureCollection - mean
reducer('ee.Reducer.mean()') - MODIS spatial resolution of 250 m

`var extracted = ndvi_scaled_2017.reduceRegions({ collection: points, reducer: ee.Reducer.mean(), scale: 250, });`

### Step 8: Export the csv table

Extracted site-level NDVI values were exported from Google Earth Engine
as a CSV file for subsequent occupancy analysis in R.

`Export.table.toDrive({ collection: extracted, description: 'NDVI_2017', fileFormat: 'CSV' });`

## Ecological Relevance

Normalized Difference Vegetation Index (NDVI) was used as a proxy for
vegetation productivity and greenness in each grid sites. This
productivity in turn gave information on the prey density and prey
habitat suitability that would influence the carnivore occupancy.
