# Preparation of Forest cover and forest loss covariates in QGIS

## Objectives

To derive yearly forest cover and forest loss data from Hansen Global Forest Change dataset in order to use them as site covariates in multiseason occupancy analysis. Covariates were generated for: 2017, 2018 and 2021.

## Data Source

**Hansen Global Forest Change** = Forest cover and forest loss rasters were obtained from - Hansen Global Forest Change dataset - Global Forest Watch

Dataset: [Global Forest Change](https://glad.earthengine.app/view/global-forest-change#bl=off;old=0;dl=2;lon=72.47546994639413;lat=22.782603340667542;zoom=5)

![Deriving data from gfc](D:/R_projects/multiseason_occupancy/output/figures/FC_Deriving%20data%20from%20gfc.png)

**Input vector layers:**

-   sample_sites.gpkg
-   study_area.gpkg

## Workflow Overview

The workflow included

1.  Importing Hansen forest raster layers

2.  Re-projecting raster and vector layers

3.  Reconstructing yearly forest condition

4.  Calculating forest loss

5.  Creating site-year covariate for occupancy analysis

## Workflow Steps:

### Step 1: Import Spatial layers

The following layers were imported into QGIS:

|         Layer         |      Location       |        Description        |
|:---------------------:|:-------------------:|:-------------------------:|
|   sample_sites.gpkg   |         gis         |   Camera deployed grids   |
|    study_area.gpkg    |         gis         |    Study area boundary    |
| treecover2000_raw.tif | data/raw/hansen_gfc | percent tree cover raster |
|   lossyear_raw.tif    | data/raw/hansen_gfc |  forest loss year raster  |

The figure below shows the study area in OpenStreetMap base layer and the sample_sites where the camera traps were deployed in all three layers

![Study area](D:/R_projects/multiseason_occupancy/output/maps/Study%20area.png)

![Sample sites](D:/R_projects/multiseason_occupancy/output/maps/sample_sites.png)

### Step 2: Reproject all layers

All raster and vector layers were reprojected to a common coordinate reference system EPSG:32644-WGS 84/UTM zone 44N using:

Raster - Projections - Warp (Reproject) and Vector - Data Management Tools - Reproject layer

A common projected CRS was required to ensure accurate distance calculations

### Step 3: Clip the raster files to the study area

The raster files were clipped to the study area using

Raster - Extraction - Clip Raster by Mask Layer, where

Input = projected_raster_layer, Mask layer = study area, resolution = 30

**Output:** forestcover2000.tif and lossyear.tif

The forestcover2000 layers gives the percentage forest cover per pixel with numbers ranging from 0 to 100. Here, 0 means no forest and 100 means dense forest. The lossyear layer tells the year of forest loss in each pixel. For ex. 2 means loss in 2002 and 16 means loss in 2016.

### Step 4: Reconstruct forest cover for each year

The Hansen data set provides baseline tree cover for the year 2000 and annual forest loss information. Year-specific forest cover layers were reconstructed by removing pixels representing forest loss occurring before each target year. For 2017: Raster calculator was used to exclude pixels with forest loss before 2017.

Raster - Raster calculator - In expression: `IF( ("lossyear@1" = 0 OR "lossyear@1" > 17), "forestcover2000@1, 0 )` **Output:** fc_2017.tif

Similarly, For 2018: `IF( ("lossyear@1" = 0 OR "lossyear@1" > 18), "forestcover2000@1", 0 )` **Output:** fc_2018.tif

And, For 2021: `IF( ("lossyear@1" = 0 OR "lossyear@1" > 21), "forestcover2000@1", 0 )` **Output:** fc_2021.tif

![Calculating forest cover for 2017](D:/R_projects/multiseason_occupancy/output/figures/FC_Calculating%20forest%20cover%20for%202017.png)

### Step 5: Calculate percentage forest cover per site

Percentage of forest cover for each site was calculated using zonal statistics where

Input = sample_sites.gpkg

Raster layer = fc_2017.tif

Statistics = Mean

**Output** = per_fc_2017.gpkg, per_fc_2018.gpkg, per_fc_2021.gpkg

We repeat this step for 2018 and 2021 as well.

The figure below shows the reconstructed forest cover around camera trap sites for 2017.

![Calculating forest cover for 2021](D:/R_projects/multiseason_occupancy/output/figures/FC_Calculating%20percentage%20forest%20cover%20for%202021.png)

![Forest Cover 2017](D:/R_projects/multiseason_occupancy/output/maps/forest_cover_2017_map.png)

### Step 6: Calculate cumulative forest loss for each year

Forest loss up to the year of our interest in each pixel was calculated using

Raster - Raster calculator - In expression: `("lossyear@1" > 0 AND "lossyear@1" <= 17)`

**Output:** loss_2017.tif, loss_2018.tif, loss_2021.tif

This step converts the Hansen loss year into binary cumulative loss raster where o means no forest was lost in that pixel and 1 means forest loss occurred between 2001 and 2017 or our year of interest.

The figure below shows the cumulative forest loss for year 2021.

![Forest Loss 2021](D:/R_projects/multiseason_occupancy/output/maps/forest_loss_2021_map.png)

### Step 7: Calculate total pixels lost per site

Total number of pixels lost in a site in certain year was calculated using zonal statistics, where

Input = sample_site.gpkg

Raster layer = loss_2017.tif

Statistics = SUM

**Output:** fl_2017.gpkg, fl_2018.gpkg, fl_2021.gpkg after repeating the steps for loss_2018 and loss_2021.

### Step 8: Calculate total raster pixels inside each site

The total number of pixels in each site was calculated using Zonal statistics where,

Input = sample_sites.gpkg

Raster layer = forestcover2000.tif

Statistics = COUNT

**Output:** count.gpkg

This file simply gives the total number of pixels in each site so that we can calculate percentage forest loss in those sites

### Step 9: Join all layer

The percentage forest cover and forest loss layers were joined into sample_sites.gpkg using siteID.

![Joining layers in sample site.gpkg](D:/R_projects/multiseason_occupancy/output/figures/FC_Joining%20the%20layers%20in%20sample_site.png)

### Step 10: Calculate percentage forest loss

The percentage forest loss for each year was calculated using

sample_sites.gpkg - attribute table - Raster - field calculator expression: `("fl_2017"/"count") \* 100`, where

**Output:** per_fl_2017 column

Output field type: Decimal

Save the edits.

![Table with covariate data](D:/R_projects/multiseason_occupancy/output/figures/FC_Attribute%20table%20of%20sample%20site%20after%20calculating%20percentage%20forest%20loss.png)

### Step 11: Export the merged file into csv format

The sample_site.gpkg layer was exported as a csv file in output/forestcover_loss.csv.

Final site covariates were exported as csv file for integration with occupancy datasets in R. In the csv file, the column names were renamed and unnecessary columns were deleted to make the forestcover_loss covariate sheet.
