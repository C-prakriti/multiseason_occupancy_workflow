# Preparation of Encounter history dataframe for Multiseason Occupancy Analysis

## Objectives

To prepare species specific multi-year encounter history matrices for
multiseason occupancy analysis using camera trap detection data.

## Inputs

| File                 | Description                |
|----------------------|----------------------------|
| camera_station.csv   | Camera deployment metadata |
| img_metadata_raw.csv | Raw image metadata         |

## Software Used

-   R
-   QGIS
-   MS Excel

## Workflow Overview

-   Extracted camera trap metadata
-   Created unique ids i.e. siteID for each grid
-   Metadata was parsed and linked to sampling location using unique
    siteID
-   Detections aggregated into 5-day samplign occasion to generate
    encounter history matrices

## Detailed Workflow

### Step 1: Data Preparation

The information of camera trap deployment for all years were maintained
into a single sheet in camera_station.csv. The camera trap images were
sorted into species specific folder and the images were renamed using
exifr into a format gridID_date_camID.

Output: raw/camera_station.csv, raw/img_metadata_raw.csv

### Step 2: Data Parsing

In the img_metadata_raw.csv file, the gridID was extracted from the
Filename, the character datetime was converted into datetime format, the
year column was created from datetime original, the sourcefile column
was removed and datetimeoriginal was renamed to datetime using R
(scripts/002.data_parsing.R)

Output: processed/img_metadata.csv

### Step 3: Create unique siteID

Each site(grid) were given a unique siteID in QGIS using,
study_area.gpkg - attribute table - field calculator where name =
siteID, type = Text, length = 10, Expression =
'ST'\|lpad(\@row_number,3,'0')

Output: A column with siteIDs in the study_area shapefile

### Step 4: Assign siteIDs to camera stations

The siteIDs were assigned to the respective camera stations using
processing toolbox - Join attributes by location - join features in:
camera_station.gpkg features: within by comparing to: study_area.gpkg
fields to add: siteID

Output: processed/cam_station.csv

### Step 5: Connect siteIDs to metadata

The siteIDs were next connected to the metadata sheet using left_join
function in R and grouping with gridID and year
(scripts/003.assigning_siteID)

Output: processed/detection_metadata.csv

### Step 6: Create encounter history matrix

To create the encounter history matrix from the cam_station.csv and
detection_metadata.csv, the datetime, deploye date and ended date in
these sheets were converted to proper datetime format. Then occassions
of 5 days were created and assigned detections to them. Then, all
possible occasions in each site_year were created using deploy date and
ended date. Based on the deploy and ended date, the camera's active
status was determined. Then the cells were filled with detection (1, 0,
NA) where, 1 = species detected in that occasion, 0 species not detected
in that occasion and NA = camera not active in that occasion.
(scripts/004.eh.R)

Output: output/encounter_history.csv

## Final Outputs

-   encounter_history.csv
