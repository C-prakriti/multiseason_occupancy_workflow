# Preparing Terrain Ruggedness Index (TRI) covariate for Multiseason Occupancy Analysis

## Objectives

To derive a site-level Terrain Ruggedness Index (TRI) covariate from
Shuttle Radar Topography Mission (SRTM) dem data to use as a covariate
in multiseason occupancy analysis.

## Data Source

Digital Elevation Model(DEM) data were obtained from the:
<https://earthexplorer.usgs.gov/> Earth explorer platform usign the SRTM
1 Arc-second global dataset, Spatial resolution: 30 m Format: GeoTIFF
CRS after reprojection: EPSG:32644 (WGS 84/ UTM Zone 44N)

## Software used

-   QGIS
-   SRTM

## Workflow Overview

-   Download SRTM DEM tiles from USGS EarthExplorer
-   Reproject DEM tiles to UTM Zone 44N
-   Merge DEM tiles into a single raster
-   Clip DEM to the study area
-   Compute Terrain Ruggedness Index (TRI)
-   Visualize and inspect terrain heterogeneity
-   Extract mean TRI values for each sampling grid using zonal
    statistics
-   Export site level TRI covariates as CSV

## Detailed Workflow

### Step 1: Download DEM Data

SRTM 1 Arc-Second Global DEM tiles were downloaded from the USGS
EarthExplorer portal. - Open USGS EarthExplorer - Define the study area
using polygon section - Navigate to: Data sets - Digital Elevation -
SRTM - SRTM 1 Arc-Second Global - Inspect DEM tile footprints covering
the study area - Download all required GeoTIFF DEM tiles

### Step 2: Reproject and merge DEM tiles

The downloaded DEM tiles were reprojected to a EPSG:32644 (WGS 84/ UTM
Zone 44N) to ensure terrain calculations were performed in metric units
using, QGIS - Raster - Reprojection - Warp(ReProject) Input: DEM tile
Target CRS: EPSG:32644 (WGS 84/ UTM Zone 44N) Resampling method:
Bilinear Output: Reproject_DEM

If your study area falls in more than one DEM tiles then they should be
merged into a single continuous raster covering the study area. Raster -
Miscellaneous - Merge Output data type: Float32 Uncheck: Place each
input file into a separate band Output: merged_dem.tif

Here, Float32 was used to preserve the decimal precision during raster
processing.

### Step 3: Clip DEM to study area

The merged DEM raster was clipped to study area boundary to reduce file
size and restrict terrain analysis to the area relevant for occupancy
modeling. Raster - Extraction - Clip Raster by Mask Layer where, Input:
merged_dem.tif Mask layer: study_area.gpkg Output: clip_dem.tif

### Step 4: Compute Terrain Ruggedness Index (TRI)

TRI was calculated from clipped DEM raster to quantify topograhic
heterogeneity across the landscape. Processing Toolbox - Terrain
Ruggedness Index - Input: clip_dem.tif Output: TRI.tif ![Terrain
Ruggedness
Index](D:/R_projects/multiseason_occupancy/output/maps/TRI.png)

### Step 5: Extract Site-level TRI covariate

A fine-scale TRI raster was converted into a site level covariate using
zonal statistics using Processing Toolbox - Zonal statistics Input:
Sample_sites.gpkg Raster layer: TRI.tif Statistics to Calculate: MEAN
Output: TRI.gpkg ![Extracting
TRI](D:/R_projects/multiseason_occupancy/output/figures/Extracting%20TRI.png)

### Step 6: Export the TRI value

The TRI value for the sample sites were exported as a csv file using
TRI.gpkg - Export - Export feature as \_ Output: Tri_cov.csv
